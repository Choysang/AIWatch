## 锁定的设计决策

| # | 决策 | 理由 |
|---|---|---|
| **P1** | 个性化 = 新增**「推荐」mode**（第三档，与 最新/精选 并列），**不重排默认的 最新 流** | 最新流是严格时间序（用户在 60768e8/30cd699 专门修复过，不能回退）。推荐是显式 opt-in 视图，「千人千面的首页」由这一档承载。 |
| **P2** | 读者画像**查询时算、不持久化** | owner 画像可预算进 `events.rank_score`（全局一份）；读者有千万 rid 身份，无法逐个预算。读者信号少、画像小，TS 聚合 + 候选集重排足够快。 |
| **P3** | 候选池 = 近窗（~60 天）取 top-N（~150）→ **TS 重排**（非 SQL 注入） | 推荐要新鲜（不翻陈年高分）。候选有上限，纯 TS 重排 over ≤150 张卡片极快、低风险（owner 走 SQL 是因为要 UPDATE 全表，A3 不持久化）。 |
| **P4** | 信号**加权**：star +3 / like +2 / view +0.5 / down −4 | view 极多且弱，不能与 like/star 等权（否则浏览淹没偏好）。down=显式「不感兴趣」，强负。 |
| **P5** | boost 维度含 **tag**（owner 的 boost 只用 source/category/contentType） | 读者偏好的核心载体是 tag（主题板就是 tag），且卡片已带 tags。 |
| **P6** | 读者**显式 down 过的事件**：在推荐里**整条排除** | 已说「不感兴趣」，不该再推荐。boost 只惩罚同类，具体那条直接隐藏。 |
| **P7** | 冷启动（无信号/空画像）→ 回退时间序 | 新读者推荐档不该空白；回退成近窗时间序，照样可用。 |

## 模型（src/scoring/reader-affinity.ts，纯函数）

```
ReaderSignal = { signal: "star"|"like"|"view"|"down"; tags[]; sourceId; category; contentType }

buildReaderAffinityProfile(signals): 每 (dim, key) 累加
  weightedScore += W[signal]   (W = star 3, like 2, view 0.5, down -4)
  n += 1
  affinity(dim,key) = n >= minSamples ? clamp(weightedScore / SATURATION, -1, +1) : 0
  → Profile { tag, source, category, contentType }   (复用 owner-affinity 的 AffinityTable 形状)

computeReaderBoost(eventDims, profile, config):
  tagAff = mean(profile.tag[t] for t in event.tags)        // tag 维度取均值
  mean   = avg over present dims of { tagAff, source, category, contentType }
  return clamp(affinityBoostMax * mean, ±affinityBoostMax)

readerAffinityConfig = { weights, saturation≈6, affinityBoostMax≈25, minSamples≈2, version "reader-affinity-v1" }
```
`affinityBoostMax≈25`：相对 qualityScore(0–100) 能有效重排但不碾压质量。

## 数据（src/db/queries/reader-affinity.ts）
`loadReaderSignals(identity)`：按 XOR 身份（userId/fingerprint，同 reactions）读
- `event_reactions`（like/star/down）⨝ events 取 tags/main_source_id/category/content_type
- `event_views`（view）⨝ events 同上
返回 `{ signals: ReaderSignal[]; downedEventIds: string[] }`。

## 推荐流（A3.2）
- `PublicMode += "personalized"`；`parsePublicQuery` 认 `mode=personalized`。
- `searchPersonalized(identity, filter, limit)`：
  1. `candidates = searchEvents({...filter, mode:"all", since: 近窗}, POOL=150)`
  2. `{signals, downedEventIds} = loadReaderSignals(identity)`；`profile = build...`
  3. 空画像 → `candidates.slice(0, limit)`（冷启动回退，P7）
  4. 否则：排除 downedEventIds（P6）→ 每张算 `score = base(qualityScore) + computeReaderBoost(card, profile)` → 稳定降序（同分按有效时间）→ top `limit`
- 首页 `mode=personalized` 时走 `searchPersonalized`（身份来自 cookie/session，复用 reader-identity）。
- search-bar 加「推荐」tab；i18n `search.mode.personalized`。
- 主题板「进入」可带 `mode=personalized` → 板内也个性化（可选，A3.2 末尾）。

## 风险
- 匿名读者信号稀疏 → 多数走冷启动回退（可接受；登录/活跃读者收益最大）。
- 候选池近窗可能漏掉读者偏好的老事件（取舍：新鲜 > 全覆盖；可调窗口）。
- base 用 qualityScore 而非 rank_score（卡片未带 rank_score）；qualityScore 是稳定的质量基线，足够。
