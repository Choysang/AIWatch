# v0.5 切片一 · A1 主题板 + A4 OPML/自助订阅 — 设计

> 上游：[v0.5-diy-plan.md](./v0.5-diy-plan.md)（三档 DIY 升级）。本文是用户锁定的 **第一刀 = A1 + A4** 的落地设计。
> 哲学不变：确定性优先 —— 主题板用 `tags[]` 数组重叠确定性匹配，不引入 embedding（那是后续 I1）。

## 锁定的设计决策

| # | 决策 | 取舍 |
|---|---|---|
| **D1** | 主题板归属 = **XOR 身份**（登录读者→`user_id`，匿名读者→`rid` cookie 的 `fingerprint`），沿用 `event_reactions`/`event_views`/`event_comments` 既有模式 | 匿名板是**设备本地**的：清 cookie 即丢、跨设备不同步；匿名→登录**不迁移**。代价可接受（与点赞/收藏一致），且登录读者天然拿到持久板。要持久跨端必须登录，而登录目前还依赖 prod 的 `RESEND_API_KEY`/`AUTH_EMAIL_FROM`（OTP），故 v1 默认匿名即可用、零注册摩擦。 |
| **D2** | A4「自助订阅」= OPML **导入即批量"信源建议"**（`contributions.kind=source_recommendation`，状态 `submitted`），主理人审核后才进全局池 | 尊重既有"主理人精选单一池"边界（[[project-data-sharing-boundary]]），复用 347ed51 的小时级 digest。**不是**让读者把任意 feed 直接灌进自己的信息流（那需要 per-reader 抓取/事件，是另一个量级且偏离产品定位）。已在 v0.5-plan A4 行写明、用户已认可。 |
| **D3** | OPML **导出** = 主理人精选池里**具备公开 feed URL 的信源**（rss 家族：`rss`/`youtube_rss`/`huggingface`），按 `categories` 分组 | rsshub/x/github 等内部连接器无公开 xmlUrl，导出无意义 → 仅导出可订阅项，诚实标注"可订阅 RSS 信源"。 |
| **D4** | 主题板 feed 复用 `searchEvents({ tags: board.tags })`（`feed.ts` 已支持 `arrayOverlaps(events.tags, tags)`） | 零新查询逻辑，board 视图天然继承时间序 + 精选/最新双模 + 既有卡片。 |
| **D5** | 板 tag 选择器 = **热门标签建议**（新 `listPopularTags()`：`unnest(tags)`+count）+ **自由输入** | 现状无 tag 词表端点（tag 是涌现的，仅卡片 chip 设过 `?tags=`）。给真实词表 + 兜底自定义。 |

## 数据模型 — `topic_boards`（迁移 0027）

```
topic_boards (
  id            text primary key,                       -- prefixed-ULID 'tb_…'
  user_id       text references user(id) on delete cascade,   -- XOR
  fingerprint   text,                                          -- XOR（rid payload）
  name          text not null,                          -- ≤40 字
  emoji         text,                                   -- 可选，单字形（板封面）
  tags          text[] not null default '{}',           -- ≤20 个；空=匹配全部（等于"全部动态"快照）
  sort_order    integer not null default 0,             -- 读者自排
  created_at    timestamptz not null default now(),
  updated_at    timestamptz not null default now()
)
```

SQL-only 约束（drizzle 无法表达 COALESCE 唯一性，沿用 event_reactions 注释惯例）：
- `CHECK (num_nonnulls(user_id, fingerprint) = 1)` — 严格 XOR。
- 每身份**板名唯一**：`unique (user_id, lower(name)) where user_id is not null` + `unique (fingerprint, lower(name)) where fingerprint is not null`。
- 列表索引：`(user_id, sort_order)` / `(fingerprint, sort_order)`。
- **同迁移补 `events.tags` 的 GIN 索引**：`create index events_tags_gin_idx on events using gin (tags)` —— 板的 `tags && board.tags` 重叠此前无索引、会顺扫。

匿名滥用护栏（应用层）：每身份板上限 **30**、每板 tag 上限 **20**、name ≤40、emoji 取首个字形；写端点按 IP `publicLimiter` 限流。

## 风险 / 已知缺口
- **D1 设备本地**：匿名板易失。后续若上读者账号体系，可加「匿名板 claim 到账号」迁移。
- **OPML 导出覆盖面**：仅 rss 家族；社交/rsshub 源不在内（诚实标注）。
- **导入限流**：单次 50 feed 上限 + IP 限流，防 OPML 灌库。
- **tag 词表**：`listPopularTags` 随数据演化；空库时选择器仅留自由输入。
