# v0.5 — 个人定制化 DIY 升级计划

> 方向：把 AIWatch 从「主理人精选的单一信息流」扩展为「读者可自定义的个人化资讯工作台」。
> 借鉴 [Syntopica](https://github.com/ZanebonoAlter/Syntopica) 的语义主题板 + 个性化策展模型，
> 但保留 AIWatch 的确定性评分哲学（LLM 只产出不可变输入，派生分由 SQL 计算）。

## Syntopica 功能 → AIWatch 现状映射

| Syntopica 能力 | AIWatch 现状 | 增量 |
|---|---|---|
| 用户自定义语义主题板 | 只有全局标签/分类 | **全新**（Tier 1 核心） |
| AI 建议新板（create/merge/skip） | 无 | 全新，依赖 embeddings |
| 阅读行为 → 偏好打分 | 有 event_views + 点赞/收藏/不感兴趣 + rank-v4 | 扩展为「每人」 |
| AI 模型路由（多模型分配主备） | **后端已完成**（routing.ts + 定价表） | 只差 UI |
| 内容分层切换（原文/全文/AI） | 原文 HTML + AI 摘要 | 加全文抽取 + 切换 |
| 每主题每日叙事简报 | 全局日/周/月报 | 下沉到主题板 |
| 主题关系图谱（人物/事件/关键词） | 判定已抽取 people/tools/tags | 加可视化 |
| OPML 导入/导出 | 无 | 全新 |
| 队列/调度监控 | graphile + _admin 健康面板 | 加面板 |
| Embedding 语义匹配 | 纯 SQL + LLM 判定，无向量 | 全新基础设施 |

## 三档拆解为可独立交付的切片

### Tier 1 · DIY 个性化核心
- **A1 主题板数据模型 + CRUD**：`topic_boards(user_id, name, tags[], created_at)`；读者建/改/删自己的「关注主题」。纯标签匹配，无 embedding 依赖。
- **A2 内容归板**：先用 tag/关键词确定性匹配（与评分哲学一致）；接口预留向量匹配以便 I1 后增强。
- **A3 个性化排序**：在 rank-v4 之上叠加每用户的 boost（复用 event_views + 点赞/收藏/不感兴趣信号）。
- **A4 OPML 导入导出 + 自助订阅**：承接已上线的 `/recommend-source` + 小时级收集摘要（347ed51）。
- **A5 AI 建议新板**（依赖 I1）：分析订阅内容，给 create/merge/skip 三档置信度建议。

### Tier 2 · 阅读体验
- **B1 内容分层切换**：原文 / 全文抽取（readability 或 Firecrawl 式）/ AI 结构化，读者自选。
- **B2 每主题叙事简报**：把日/周/月报生成下沉到主题板范围。
- **B3 主题关系图谱**：把判定里的 people/tools/events 画成可交互节点图。

### Tier 3 · 运营/power 工具
- **C1 模型路由 UI**（最便宜的快赢，后端已就绪）：多模型按能力分配 + 主备 + 切换。
- **C2 队列/调度监控面板**：graphile 任务状态、失败重试、手动触发。
- **C3 每源配置 UI**：抓取间隔、保留条数、AI 摘要开关。

### 基础设施
- **I1 Embeddings + pgvector**：经网关接 embedding 模型 + 向量列 + 回填。是 A2 自动归板与 A5 的前置。

## 推荐排期（按依赖与性价比）

1. **C1 模型路由 UI** — 无依赖、后端已完成、立竿见影，作为 v0.5 第一刀热身。
2. **A1 + A4 主题板 + OPML/自助订阅** — DIY 中枢，纯标签实现，不等 embedding。
3. **A3 个性化排序** — 复用现有信号，巩固「千人千面」。
4. **I1 Embeddings + pgvector** — 打基础。
5. **A2 自动归板 + A5 AI 建议** — 依赖 I1。
6. **B1 / B2 / B3 阅读体验** — 相互独立，可穿插。
7. **C2 / C3 运营面板** — 独立，随时补。

每一刀独立过 gate（tsc + 单测 + 集成 + build）独立提交，按既有 vX.Y.Z tag 流程发布。
